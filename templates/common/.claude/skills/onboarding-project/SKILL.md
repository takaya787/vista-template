# Onboarding Project

Checks and updates per-project business context stored in `.vista/profile/project.json`.
Initial setup is handled by the Electron app. Claude handles state detection and partial updates.

## Design Principles

- **Per-project, not global** — Unlike `me.json`, `project.json` is unique to each repository and is never symlinked. Opening a different folder means a different project context.
- **App-driven input** — All initial data collection is handled by the Electron app UI, which renders `references/project-interview-protocol.json` and writes `project.json`, `project.md`, and updates `CLAUDE.md` directly. Claude does not collect input for new projects.
- **Claude handles updates only** — After initial setup, Claude can update specific fields conversationally via `AskUserQuestion`.
- **Minimal required fields** — Only company name, industry, description, work domain, products, and currentInitiatives are required.

## Data Sources

- `.vista/profile/project.json` — Per-project profile (NOT symlinked). `isOnboardingCompleted` field indicates setup status.
- `.vista/profile/project.md` — Generated context document (auto-generated by App; do not hand-edit)
- `references/project-interview-protocol.json` — Question definitions rendered by the Electron app
- `references/project-generation-guide.md` — Rules for generating project.md from project.json

## Tone & Style

Use plain, friendly language. Avoid technical terms. Describe actions by their effect ("I'll update your project context") not mechanism ("Writing to project.json").

## Steps

### Step 1: Check Existing State

Read `.vista/profile/project.json` and check `isOnboardingCompleted`:

- **File does not exist** or **`isOnboardingCompleted` is `false` or missing** → tell the owner to open the Electron app:
  > 「このプロジェクトの業務コンテキストがまだ設定されていません。アプリを開いて「プロジェクト設定」から入力してください。」
  Then exit skill.
- **`isOnboardingCompleted: true`** → show a one-line summary and ask if update is needed:
  > 「{company.name}」の業務コンテキストが登録されています（担当: {myWork.domain}）。更新しますか、それとも続けますか？
  - "続ける" → exit skill immediately
  - "更新" → proceed to Step 2

### Step 2: Update Flow (partial updates only)

When the owner wants to update specific fields:

1. Display current values for the fields they want to change
2. Collect only the changed fields via `AskUserQuestion` (1–4 questions per call, 2–4 options per question)
3. Confirm the changes with the owner before writing
4. Write the updated fields to `project.json` (merge; do not overwrite unchanged fields)
5. Regenerate `project.md` from the merged JSON — see `references/project-generation-guide.md` for template and rules

**Completion message:**
> 「{company.name}」のプロジェクトプロフィールを更新しました。

---

> **Note:** Initial project setup (creating project.json, generating project.md, updating CLAUDE.md) is handled entirely by the Electron app. See `references/project-interview-protocol.json` for the question definitions the app renders, and `references/project-generation-guide.md` for generation rules.
