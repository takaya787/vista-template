# Documentation

After making changes to the project (adding features, changing tools, updating configuration, etc.), always update `README.md` and other relevant documentation to reflect the changes.

## Document Output Rules

- Save all documents in the format `docs/{folder_name}/{file_name}.md`
- Do not place files directly under `docs/` (exception: definition files like `team.md`)
- Folder names use English kebab-case describing the purpose (e.g., `weekly-updates`, `planning`, `sprint-goals`)
- File names should have a date prefix: `{YYYY-MM-DD}-{description}.md`
- All AI-generated documents are gitignored (`docs/` is not tracked by Git)
- Only definition files (`team.md`, `members/*.md`) are tracked by Git

## Asset Output Rules

- Save screenshots, videos, and other media generated by Playwright etc. to `assets/`
- Files are organized into subdirectories by type:

```
assets/
├── screenshots/     # Screenshots (.png)
├── videos/          # Videos (.webm)
└── traces/          # Playwright traces (.zip)
```

- File name format: `{YYYY-MM-DD}-{description}.{ext}`
  - Example: `2026-03-01-sprint-dashboard.png`
  - Example: `2026-03-01-notion-standup.webm`
- Timestamp-only file names (e.g., `screenshot-1772159629316.png`) are prohibited
- The entire `assets/` directory is gitignored (generated artifacts, no Git tracking needed)
