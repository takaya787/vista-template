# Documentation

After making changes to the project (adding features, changing tools, updating configuration, etc.), always update `README.md` and other relevant documentation to reflect the changes.

## Document Output Rules

### MANDATORY: All documents MUST be saved under `docs/`

**AI-generated documents placed in the project root or any directory outside `docs/` is a rule violation.**
This includes but is not limited to: reports, meeting minutes, plans, summaries, analyses, letters, specs, and any other text documents generated by AI.

- **Required path format**: `docs/{folder_name}/{file_name}.md`
- **Prohibited**: Creating `.md`, `.txt`, `.docx`, `.pdf`, or any document file in the project root (`./`) or arbitrary directories
- If the appropriate subfolder under `docs/` does not exist, create it
- Folder names use English kebab-case describing the purpose (e.g., `weekly-updates`, `planning`, `sprint-goals`)
- File names should have a date prefix: `{YYYY-MM-DD}-{description}.md`

### Self-Check Before Writing

Before creating any document, verify:

1. Is the output path under `docs/`? If not, **stop and correct the path**
2. Is there an appropriate subfolder? If not, create one with a kebab-case name
3. Does the filename have a date prefix? If not, add one

## Asset Output Rules

- Save screenshots, videos, and other media generated by Playwright etc. to `assets/`
- Files are organized into subdirectories by type:

```
assets/
├── screenshots/     # Screenshots (.png)
├── videos/          # Videos (.webm)
└── traces/          # Playwright traces (.zip)
```

- File name format: `{YYYY-MM-DD}-{description}.{ext}`
  - Example: `2026-03-01-sprint-dashboard.png`
  - Example: `2026-03-01-notion-standup.webm`
- Timestamp-only file names (e.g., `screenshot-1772159629316.png`) are prohibited
- The entire `assets/` directory is gitignored (generated artifacts, no Git tracking needed)
